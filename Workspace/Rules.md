# Правила рабочего пространства

Этот файл хранит рабочие правила и предпочтения.

## Правила

### Нейминг

1. Имена корневых и содержательных папок должны быть на английском языке и начинаться с заглавной буквы.
2. Технические папки, имя которых является идентификатором инструмента, пакета или навыка, могут использовать требуемый формат, например `kebab-case` для папок внутри `Skills`.

### Язык

3. Все скиллы и инструкции должны быть написаны на русском языке.

### Режим спора

4. Режим спора по методологии хранится в отдельном файле: [Methodology-Debate.md](../Agents/Methodology-Debate.md).
5. Отдельные агенты режима спора хранятся в файлах: [Critic.md](../Agents/Critic.md) и [Integrator.md](../Agents/Integrator.md).

### Хранение правил

6. Не создавать отдельный файл для каждого правила без необходимости.
7. Если правило новое, записывать его в этот файл.
8. Если правила можно логично сгруппировать, объединять их в один раздел.
9. Если новое правило относится к уже существующей группе правил, добавлять его в эту группу.
10. Не складывать тематические документы в `Workspace`, если для них уже есть отдельная корневая папка.
11. Использовать `Workspace` для правил, контекста, решений и вспомогательных указателей, а не для основного содержания тематических разделов.
12. Хранить предметные знания в `Knowledge`, рабочие процессы в `Skills`, роли анализа в `Agents`, материалы инициатив в `Projects`, внешние исходники в `Sources`.
